# HTML-ppt 提示词模板

用于给 `HTML-ppt` skill 提供稳定、可执行的输入。  
目标不是只描述“想做一份 PPT”，而是一次性给出足够信息，让 skill 能顺畅完成：

`业务材料 -> HTML 网页 PPT -> 导出 PPTX -> 交付 html + pptx`

## 最少要给什么

至少提供这 3 项：

1. `主题`
这份 PPT 讲什么。

2. `受众`
给谁看。

3. `页数目标`
希望做成多少页，建议 `8-12` 页。

如果只给这 3 项，skill 可以开工，但生成质量会更依赖默认假设。

## 每一项怎么写

`项目名称`
- 一句话写清主题。
- 例：`智能客服平台升级方案汇报`

`使用场景`
- 说明这份 PPT 的使用环境和目标。
- 例：`面向公司管理层做 15 分钟汇报，目标是申请 200 万预算启动项目一期。`

`目标受众`
- 尽量具体到角色。
- 例：`CEO、COO、客服负责人、技术负责人`

`页数要求`
- 建议直接给范围。
- 例：`控制在 8-10 页`

`风格要求`
- 描述视觉和表达方式。
- 例：`Swiss Style，专业、克制、现代，强调留白、网格和重点数字。`

`配色要求`
- 写清主色和禁用色。
- 例：`商务风，深蓝为结构底色，橘色做重点强调，避免紫色。`

`品牌要求`
- 写 logo、品牌名、字体、比例等限制。
- 例：`首页需要放公司 Logo，整体保持 16:9。`

`核心信息`
- 写清最重要的 3-6 个事实或观点。
- 不要先写“目录”，直接写结论和内容。

`关键数据`
- 优先给可上页的数字。
- 例：`月均咨询量 82 万，重复问题占比 57%，单月人力成本约 138 万。`

`希望包含的页面`
- 告诉 skill 你想看到哪些模块。
- 例：`背景、问题、方案、收益、实施计划、风险控制、结尾 takeaways`

`实施计划 / 时间线`
- 最好按季度或阶段写。
- 例：`Q1 知识库治理，Q2 FAQ 机器人上线，Q3 坐席辅助，Q4 评估扩展。`

`风险与对策`
- 用“风险 -> 对策”形式最清楚。
- 例：`知识库质量不足 -> 先覆盖高频问题并建立更新机制。`

`必须强调的一句话`
- 用来做封面、副标题或结尾页。
- 例：`这不是裁员项目，而是提升响应效率和服务体验的升级项目。`

`硬性约束`
- 任何不能违背的要求都提前写。
- 例：
  - `必须导出 html + pptx`
  - `不能使用紫色`
  - `需要中英双语`

`附件素材`
- 列出现有文档、数据、旧版 PPT、截图、图片素材。

## 推荐写法

- 用“结论 + 事实”表达，不要只写抽象方向。
- 优先给数字、对比、阶段、目标。
- 如果有旧 PPT、文档、图表，直接一起提供。
- 如果没有完整内容，也至少给主题、受众、场景、页数和风格。

## 不推荐写法

- 只说：`帮我做一份高大上的 PPT`
- 只说：`做个商务风`
- 只给标题，不给受众和场景
- 给很多背景资料，但不说这份 PPT 最终想达成什么目标

## 可直接使用的示例

```md
项目名称：
智能客服平台升级方案汇报

使用场景：
面向公司管理层做 15 分钟汇报，目标是申请 200 万预算启动项目一期。

目标受众：
CEO、COO、客服负责人、技术负责人

希望输出：
1. 一个可横向翻页展示的 HTML 网页 PPT
2. 一个可发送给管理层归档的 PPTX 文件

页数要求：
控制在 8-10 页

风格要求：
Swiss Style，专业、克制、现代，少装饰，强调网格、留白和重点数字。

配色要求：
商务风，深蓝为结构底色，橘色做重点强调，避免紫色。

品牌要求：
首页需要放公司 Logo，比例固定 16:9。

核心信息：
1. 客服响应慢，平均首次响应时间 4.2 分钟。
2. 高峰期排队严重，用户满意度下降。
3. 重复问题占比高，人工成本持续上升。
4. 计划引入 AI 意图识别、知识库问答、工单自动分派、坐席实时辅助。
5. 目标是在不牺牲体验的前提下，降低重复劳动并提升服务效率。

关键数据：
1. 月均咨询量：82 万
2. 人工解决率：91%
3. 重复问题占比：57%
4. 当前客服团队人数：86 人
5. 单月人力成本：约 138 万元

希望包含的页面：
1. 背景与问题
2. 方案架构
3. 核心能力模块
4. 收益测算
5. 实施计划
6. 风险与治理
7. 收尾页

实施计划 / 时间线：
Q1：需求梳理与知识库建设
Q2：上线 FAQ 机器人与分流
Q3：上线坐席辅助与质检
Q4：效果评估与二期规划

风险与对策：
1. 知识库质量不足 -> 先治理高频问题
2. 员工抵触 AI -> 先做人机协同，不直接替代
3. 初期效果波动 -> 灰度上线并保留人工兜底

必须强调的一句话：
这不是裁员项目，而是提升响应效率和服务体验的升级项目。

硬性约束：
1. 最终必须交付 html + pptx
2. 不能使用紫色
3. 页面不要超过 10 页

附件素材：
1. 公司 Logo
2. 客服运营月报
3. 旧版汇报 PPT
4. 一张现有流程图
```

## 给 skill 使用者的建议

- 如果你希望生成结果更接近真实商务汇报，优先补充：
  - 目标受众
  - 一句话目标
  - 关键数据
  - 风格和配色
  - 硬性约束
- 如果你希望结果更快落地，优先补充：
  - 时间线
  - 风险与对策
  - 现有附件素材
- 如果你只是在试模板，也尽量不要少于：
  - `主题 + 受众 + 页数 + 风格 + 3 条核心信息`
